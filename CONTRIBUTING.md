# 参与贡献

感谢参与改进别离检测工具。

1. 先通过 Issue 描述缺陷或功能建议，避免重复工作。
2. 从默认分支创建短期分支，提交范围保持单一。
3. 确保 `dotnet build -c Release` 成功，并说明人工测试环境。
4. 提交 Pull Request，写明改动、原因、用户影响和验证方式。
5. 外部贡献必须由项目 Reviewer 审查后才能合并。

涉及硬件底层访问、网络下载、管理员权限、安装程序、GitHub Actions 或 SignPath 的修改需要额外安全审查。请勿提交证书、令牌、设备识别信息或真实导出文件。

提交贡献即表示你同意按 GPL-3.0-only 许可该贡献。
